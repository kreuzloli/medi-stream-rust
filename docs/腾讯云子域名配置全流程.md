# 腾讯云标准直播（LVB/CSS）子域名配置全流程（推流 + 播放）

[TOC]

## 1. 背景与目标

你当前情况：

- 已有主域名：`domain.com`
- 该域名（或 `www.domain.com`）已经解析到你自己的腾讯云服务器（用于网站/接口）
- 现在要接入腾讯云 **标准直播 LVB/CSS**，需要配置：
  - 推流域名（主播推流用）
  - 播放域名（观众播放用）

核心原则：

- **直播域名不要指向你的服务器 IP**（不要 A 记录到 CVM）
- 直播域名一般要 **CNAME 到腾讯云直播给你的目标域名**（走腾讯云的源站/CDN）
- 你的网站域名继续保持 **A 记录到服务器**，互不影响

---

## 2. 推荐域名规划

在同一个主域名下创建两个子域名：

- 推流域名：`push.domain.com`
- 播放域名：`play.domain.com`

**可选**（建议以后扩展）：

- 业务站点：`www.domain.com`（A 到服务器）
- API：`api.domain.com`（A 或 CNAME 到负载均衡）
- 回调：`cb.domain.com`（A 或 CNAME 到业务服务）

> 注意：推流域名和播放域名 **必须不同**。用两个子域名就完全满足要求，不需要买第二个主域名。

---

## 3. 前置检查清单（先过这一关，少踩坑）

### 3.1 DNS 托管位置确认

你之前 `dig NS domain.com` 返回 `*.dnspod.net`，说明 DNS 在 DNSPod（腾讯系）。
因此 **CNAME 解析记录要在 DNSPod/腾讯云 DNS 解析里配置**。

### 3.2 备案提醒

- 如果你直播服务面向 **中国大陆**，通常要求域名已备案。
- 你现在网站已经在腾讯云服务器上跑，基本也绕不开备案这个话题（国内资源）。

### 3.3 记录冲突规则（最常见翻车点）

同一个主机记录（比如 `push`）：

- **不能同时存在 A 记录和 CNAME 记录**
- 如果你之前给 `push`/`play` 误配过 A 记录指向服务器，必须先删掉再配 CNAME

---

## 4. 在腾讯云直播控制台创建域名（先“登记”，再做 DNS）

> 你先在直播控制台添加域名，系统才会给你“CNAME 目标地址”（你要把它填到 DNSPod 里）。

### 4.1 添加推流域名

路径大致是：

- 腾讯云控制台 → 云直播（CSS）→ 域名管理 → 添加域名
  填写：
- 域名：`push.domain.com`
- 类型：推流域名（PUSH）

保存后，你会看到提示类似：

- “请将 `push.domain.com` CNAME 到 `xxxxx.livepush.myqcloud.com`”

把这串 `xxxxx.livepush.myqcloud.com` 记下来，下一步要用。

### 4.2 添加播放域名

同样操作：

- 域名：`play.domain.com`
- 类型：播放域名（PLAY）

保存后会提示：

- “请将 `play.domain.com` CNAME 到 `yyyyy.liveplay.myqcloud.com`”

把 `yyyyy.liveplay.myqcloud.com` 记下来。

---

## 5. 在 DNSPod（腾讯云 DNS）配置两条 CNAME（真正“拆子域名”的动作）

进入：

- 腾讯云控制台 → DNS 解析（DNSPod/云解析）→ 选择 `domain.com` → 添加记录

### 5.1 配置推流 CNAME

新增记录：

- 主机记录：`push`
- 记录类型：`CNAME`
- 记录值：`xxxxx.livepush.myqcloud.com`（直播控制台给你的那串）
- TTL：默认即可

最终效果：

- `push.domain.com` → CNAME → `xxxxx.livepush.myqcloud.com`

### 5.2 配置播放 CNAME

新增记录：

- 主机记录：`play`
- 记录类型：`CNAME`
- 记录值：`yyyyy.liveplay.myqcloud.com`（直播控制台给你的那串）
- TTL：默认即可

最终效果：

- `play.domain.com` → CNAME → `yyyyy.liveplay.myqcloud.com`

---

## 6. 验证是否生效

在 Mac 终端执行：

```bash
dig push.domain.com CNAME
dig play.domain.com CNAME
```
